## 经验教训（血的教训）

1. **函数签名改一处查全局** — 改参数签名后 grep 所有调用点
2. **补丁阈值=3** — 同文件超3次补丁，暂停评估是否重写
3. **不改 JS 用 sed** — 用 edit 工具
4. **用户重复=关键信息** — 说超过2遍的内容一定重要
5. **镜像修复** — 修A页面bug顺手检查B/C页面
6. **交付前全流程跑一遍** — 修完走完整用户路径
7. **先教后考** — 交互问答放在内容介绍之后
8. **每页只做一件事** — 不混内容
9. **命名全称** — 正式产品名一个字不能省
10. **先拿原件再干活** — 数据以原件为准
