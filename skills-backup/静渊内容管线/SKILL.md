# 静渊内容管线 — 奶豆的 SKILL.md

> 基于 content-pipeline (OrangeViolin/01fish) 的理念改造
> 适配 OpenClaw + DeepSeek 环境
> 一句话触发，一条龙出活

---

## 触发词总览

| 你说 | 我干什么 | 输出 |
|------|---------|------|
| `出稿` | 根据最近的素材写一篇完整文章 | 公众号草稿 |
| `排版` | 把文章转成公众号阅读格式 | HTML/Markdown |
| `做头图` | 生成封面图设计描述+参考 | 头图方案 |
| `分析 [链接]` | 抓链接+深度解读意义 | 分析报告 |
| `早报` | 立即生成一期科技早报 | 当日早报 |
| `深度 [话题]` | 把一个话题写成深度文 | 完整文章 |
| `小红书 [链接/话题]` | 转成小红书轮播图文案 | 小红书内容 |
| `系列 [话题]` | 策划一个系列选题 | 选题规划 |

## 工作流 01：出稿

### 触发词
`出稿` 或 `写一篇关于 [主题] 的文章`

### 执行步骤
1. 回顾近期对话/素材/早报中相关的积累
2. 确定文章定位：教程类 / 说明书类 / 深度长文 / 趋势解读
3. 调用三道壕沟法：
   - 查5个以上来源（搜索+已知信息）
   - 写300字核心梗概
   - 设计3版开头
4. 输出全文

### 输出规范
- 标题：吸引人但不夸大，10秒内让读者判断值不值得看
- 结构：逻辑通顺，信息密度够，没有废话段落
- 风格：简洁有力，不啰嗦，用人话解释概念
- 署名格式：
  ```
  ✅ 素材收集 & 撰稿：AI（DeepSeek）
  ✅ 主导 & 审校：静渊
  ```

### 品牌色
- 主色：墨绿 (#2D5016)
- 强调色：暖红 (#C73E3A)
- 背景色：宣纸底色 (#F5F0E8)

## 工作流 02：排版

### 触发词
`排版` 或 `排一下版`

### 执行步骤
1. 接收用户提供的文章内容
2. 转换为适合公众号阅读的格式：
   - 短段落（每段不超过5行）
   - 小标题用 `##` 或 `###`
   - 引用用 `>`
   - 重点用 **加粗**
   - 列表用 `-`
   - 代码用反引号
3. 输出标准 Markdown 格式

### 输出规范
```
【标题】

正文内容...

---

📌 **今日互动：** [抛一个问题引导读者评论]
```

---

## 工作流 03：做头图

### 触发词
`做头图` 或 `生成封面图`

### 执行步骤
1. 理解文章主题和调性
2. 设计封面图方案（文字+视觉描述）
3. 输出：封面图设计稿描述

### 输出规范
```
## 封面图方案

**主标题：** [10字以内]
**副标题：** [20字以内]

**视觉设计：**
- 背景：[颜色/材质/场景描述]
- 字体：[风格描述]
- 点缀：[装饰元素建议]

**参考意境：** [一句描述图片给人的感觉]
```

## 工作流 04：分析链接

### 触发词
`分析 [URL]` 或 `看看这个 [URL]`

### 执行步骤
1. 抓取链接内容
2. 理解核心观点和数据
3. 从以下角度进行深度分析：
   - 这个信息是什么？（一句话概括）
   - 跟静渊的什么关系？
   - 能用在哪些地方？（早报/深度文/选题灵感）
   - 有什么更大的趋势信号？
   - 有没有「静渊关注」的深度选题价值？
4. 输出去向建议：存档 / 入早报 / 写深度

### 输出规范
```
## 链接分析：×××

**一句话核心：** ...

**跟你的关系：** ...

**价值判断：**
- [ ] 适合放早报（轻量信息）
- [ ] 适合写深度（有挖掘空间）→ 🔔 静渊关注
- [ ] 适合存档（以后可能用到）

**推荐行动：** ...
```

## 工作流 05：早报（即时生成）

### 触发词
`早报` 或 `今天早报呢`

### 执行步骤
1. 搜索当前科技热点（36氪、虎嗅、知乎、GitHub等）
2. 精选9条最有价值的
3. 每条配解读（观点，不要复述新闻）
4. 需要深挖的标 🔔

### 输出规范
```
【X月X日 · 周X · ☕】

📌 今日精选 9 条

**① [标题]**
🔗 来源：[网站名](链接)
💡 **解读：** ...

**② ...**

---

📊 **今日关键词：** [3-5个]
静渊，新的一天加油！
```

## 工作流 06：深度

### 触发词
`深度 [话题]` 或 `把这个展开写一篇`

### 执行步骤
1. 确定话题核心论点
2. 搜索5个以上相关信息源
3. 写300字核心梗概
4. 设计文章结构（四幕式深度框架）
5. 写全文（1000-3000字）
6. 标注 🔔 选题方向

### 四幕式深度框架
- **第一幕：问题/现象（建立共鸣）**
- **第二幕：拆解（讲清逻辑链条）**
- **第三幕：案例/对比（用事实撑观点）**
- **第四幕：趋势判断（给读者留下思考）**

## 工作流 07：小红书

### 触发词
`小红书 [话题/链接]` 或 `转成小红书`

### 执行步骤
1. 从原文提取核心要点
2. 拆成8-10张轮播图文案
3. 每张配文字+画面描述
4. 写发布文案+标签

### 输出规范
```
## 小红书轮播图

**封面：** [标题]
**内页1：** [痛点/共鸣]
**内页2-8：** [核心干货逐页展开]
**尾页：** [引导关注/收藏]

**发布文案：** ...
**标签：** #... #... #...
```

---

## 工作流 08：系列规划

### 触发词
`系列 [话题]` 或 `策划一个系列`

### 执行步骤
1. 确定大主题
2. 拆成5-10个子话题
3. 每个子话题确定：内容定位+核心论点+素材来源
4. 输出系列规划表

### 输出规范
```
## 系列选题：×××

| 序号 | 标题 | 定位 | 素材来源 |
|------|------|------|---------|
| 01 | ... | ... | ... |
| 02 | ... | ... | ... |

**总长度：** [预计多少篇/多久更新完]
**目标读者：** ...
**差异化角度：** 跟别人写的有啥不同
```
