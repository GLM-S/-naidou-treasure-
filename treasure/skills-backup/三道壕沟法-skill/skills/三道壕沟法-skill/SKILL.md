# 三道壕沟法：认知筛选与决策方法论

> 核心思想：**好东西都是总结出来的，不是猜出来的。**
> 每条规则都是**上下文相关的**。先读问题，再调参数，然后执行。

---

## 0. 问题推理（开战前先看清战场）

在任何输出之前，先**判断问题的本质**。大多数输出不好，是因为跳到了自己的默认处理模式，而非先读懂问题。

### 0.A 先读这些信号

1. **问题类型** — 写文章（科普/深度/观点）、做产品决策（方向选择/功能取舍）、分析趋势（技术判断/行业判断）、解决问题（具体执行/策略规划）
2. **目标对象** — 公众号读者（C端）、技术团队、投资人、自己（个人精进）。受众决定表达方式，不是你的偏好。
3. **时间窗口** — 当下热点（时效性高）、长期趋势（时效性低）、历史分析（已沉淀）、未来预判（不确定高）
4. **信息充足度** — 资料很多（提炼+洞察）、资料很少（反推+推理）、资料零散（拼图+假设）
5. **关键约束** — 字数限制、交付时间、可引用源、是否可公开、安全边界

### 0.B 输出一行"认知宣言"再动手

在开始任何动作前，用一句话宣称：**"我判断这是：\<问题类型\>，面向\<目标对象\>，信息充足度\<高/中/低\>，我会用第\<X\>道壕沟开始主攻。"**

示例：
- *"我判断这是：一篇科普类深度文章，面向公众号C端读者，信息充足度中（有一手资料但缺少对比案例），我会从第一道壕沟（查源）开始主攻，第三道（开头）收尾。"*
- *"我判断这是：产品方向决策分析，面向自己的判断，信息充足度低（新领域，公开资料很少），我会从第零道（值不值得干）开始，然后第二道（核心矛盾）深入。"*
- *"我判断这是：趋势分析类观点文，面向科技读者，信息充足度高（多源信息已充分交叉），我会从第三道（表达方式）开始，前三道已自然完成。"*

### 0.C 如果模棱两可，问一个问题，不要猜

只问**一个**澄清问题——绝不丢一串问题出去。只在认知宣言真正有分歧时才问。

示例：*"这篇文章的受众是偏技术背景的，还是完全不懂技术的普通读者？"*

如果能从上下文自信地推断，**不要问**。直接声明认知宣言，然后推进。

### 0.D 反默认纪律

不要默认：用「震惊体」写标题、开头先讲一长段背景铺垫、结果甩一个浅显结论、用AI常见话术写观点。这些是LLM默认，或是新手写作的默认。需要基于认知宣言，主动绕过。

## 1. 三道壕沟法的核心结构（四道壕沟 + 一道元壕沟）

每一道壕沟解决一层问题。越往后，离最终产出越近。

| 壕沟 | 名称 | 核心问题 | 产出 |
|------|------|---------|------|
| 第零道 | 值不值得干 | 这件事值得我花时间吗？ | YES/NO + 理由 |
| 第一道 | 广撒网 | 我查了够多的信息源吗？（≥5个） | 信息地图 |
| 第二道 | 扎深根 | 核心矛盾是什么？一句话讲得清吗？ | 300字核心梗概 |
| 第三道 | 开口子 | 怎么开头最有冲击力？ | 3版开头 |
| 第四道 | 交付物 | 别人能拿到什么？ | 完整交付物 |

## 2. 三组可调参数（核心配置）

执行前先设定这三组参数。每道壕沟的决策都受这些参数约束。

### 2.A 信息宽度 (INFORMATION_WIDTH) — 1到10

| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1-3 | 轻度搜索（3-5个来源，快速检索） | 日常资讯整理、捡观点、社交回复 |
| 4-6 | 中度搜索（5-10个来源，交叉验证） | 公众号常规文章、分析一个已知话题 |
| 7-8 | 深度搜索（10-20个来源，跨领域查找） | 深度长文、陌生领域创作、趋势判断 |
| 9-10 | 研究级搜索（20+来源，学术/行业报告级别） | 重大决策、新赛道判断、产品方向 |

**默认值：6**（中等深度，适合大多数内容创作）

### 2.B 深度层次 (DEPTH_LEVEL) — 1到10

| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1-3 | 浅层（信息整理、简单观点、what级） | 资讯简报、每日快讯、社交分享 |
| 4-6 | 中层（逻辑分析、why级，三道壕沟的标配） | 主流公众号深度文、观点分析 |
| 7-8 | 深层（系统性分析、how级，跨领域连接） | 趋势判断、行业洞察、方法论产出 |
| 9-10 | 顶层（元分析、模式识别、范式判断） | 决策框架设计、方法论进化、宏观判断 |

**默认值：6**（中层偏深，有洞察但不至于过于抽象）

### 2.C 风险保守度 (RISK_AVERSION) — 1到10

| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1-3 | 激进（大胆判断、敢下结论、承担打脸风险） | 个人判断、日记、实验性内容 |
| 4-6 | 中性（有观点但留余地、承认不确定性） | 公众号公开发表、产品建议 |
| 7-8 | 保守（多角度论证、充分举证、少下断言） | 投资决策、职业建议、正式交付 |
| 9-10 | 极端保守（只陈述事实、不表达观点） | 诉讼相关、政治敏感、利益冲突 |

**默认值：5**（中性偏一点安全）

### 2.D 参数推算表（问题信号 → 参数值）

| 信号 | INFORMATION_WIDTH | DEPTH_LEVEL | RISK_AVERSION |
|------|:---:|:---:|:---:|
| "写一篇公众号深度文" | 6 | 6 | 5 |
| "看一眼这个趋势" | 4 | 4 | 4 |
| "帮我分析这个赛道值不值得进" | 8 | 7 | 7 |
| "写一篇科普" | 6 | 5 | 4 |
| "给投资人看" | 7 | 7 | 8 |
| "自己琢磨用（不发表）" | 5 | 7 | 3 |
| "竞品分析" | 8 | 6 | 5 |
| "热点新闻观点" | 5 | 5 | 5 |
| "方法论/心法类" | 6 | 8 | 5 |
| "批判/争议话题" | 8 | 7 | 8 |

## 3. 信道选择指南（什么信息从哪找）

不是所有问题都用同一个信源策略。根据问题类型匹配信道：

### 3.A 科技趋势/新闻热点
| 信道 | 用途 | 优先级 |
|------|------|:------:|
| Tavily/Brave搜索 | 综合信息、多源覆盖 | ⭐⭐⭐ |
| 36氪/虎嗅 | 中文科技商业报道 | ⭐⭐⭐ |
| GitHub Trending | 开源项目风向 | ⭐⭐ |
| Hugging Face | AI模型前沿 | ⭐⭐ |
| ArXiv/论文预印本 | 学术前沿 | ⭐⭐ |
| Twitter/X / 知乎 | 行业观点、讨论热度 | ⭐⭐ |
| 公司官网/产品页 | 一手信息 | ⭐⭐⭐ |

### 3.B 历史/人文知识
| 信道 | 用途 | 优先级 |
|------|------|:------:|
| Wikipedia | 基础事实 | ⭐⭐⭐ |
| 古籍原文/注释 | 一手文本 | ⭐⭐⭐ |
| 学术论文（知网/Google Scholar） | 权威研究 | ⭐⭐⭐ |
| 专门博客/专栏 | 深度分析 | ⭐⭐ |
| 知乎/豆瓣 | 讨论和不同视角 | ⭐⭐ |

### 3.C 产品决策/商业分析
| 信道 | 用途 | 优先级 |
|------|------|:------:|
| 竞品官网/产品 | 一手功能 | ⭐⭐⭐ |
| 用户评价/论坛 | 真实反馈 | ⭐⭐⭐ |
| Crunchbase/PitchBook | 融资、估值 | ⭐⭐ |
| 行业报告 | 宏观数据 | ⭐⭐ |
| 创始人访谈/blog | 战略意图 | ⭐⭐ |
| 分析师报告 | 专业评估 | ⭐⭐ |

### 3.D 跨源验证纪律（硬规则）
1. **任何关键事实至少有两个独立信源交叉验证。** 没有交叉验证的信息标注「待核实」。
2. **区分一手源和二手源。** 公司公告 / 论文原文 > 媒体报道 > 社交讨论 > 个人推测。
3. **对立的观点也要找。** 如果所有源都说一个方向，主动搜索反方。
4. **标注置信度。** 事实（✅确凿）、推理（🟡推测）、观点（💬一家之言）。

## 4. 第零道壕沟：值不值得干

> **时间是最稀缺的资源。这道壕沟的产出只有 YES 或 NO。**

### 4.A 五问决策框架
接到任何请求，先问自己五个问题：

1. **这对我（静渊）的产品/内容方向有帮助吗？**
   - YES：帮助大不大？（核心相关/边缘相关/不相关）
2. **我现在做这件事，比做别的事回报更高吗？**
   - 考虑：时间成本、精力消耗、ROI
3. **这件事有长期价值吗？**
   - 一次性的还是可持续的？能累积成作品/能力/资源吗？
4. **是我的不可替代性吗？**
   - 还是AI能干得更好/更快？如果是，交给AI。
5. **不做会怎样？**
   - 如果一个月后回头看，这件事重要吗？

### 4.B 决策矩阵

| 方向帮助 | 长期价值 | 不可替代 | 行动 |
|:--------:|:--------:|:--------:|------|
| ✅高 | ✅高 | ✅是 | **无条件做** — 优先排期 |
| ✅高 | ✅高 | ❌非 | 让AI先做，自己审 |
| ✅高 | ❌低 | ✅是 | 快速完成，不做深 |
| ✅高 | ❌低 | ❌非 | 考虑跳过 |
| ❌低 | ✅高 | ✅是 | 少量投入，看趋势 |
| ❌低 | ✅高 | ❌非 | 存档，以后再说 |
| ❌低 | ❌低 | ✅是 | **不做** |
| ❌低 | ❌低 | ❌非 | **绝对不做** |

### 4.C 拒绝的艺术
如果答案是 NO，给出明确理由，而不是沉默/拖延。理由用一句话说清。

## 5. 第一道壕沟：广撒网

> **在没有查5个以上来源之前，不写第一个字。**

### 5.A 搜索协议
1. 确定搜索关键词（至少3组不同角度的关键词）
2. 每个关键词搜前5-10条结果
3. 快速扫读，判断信息类型（事实/观点/数据/谣言）
4. 打标签做分类
5. 标记待深挖的链接

### 5.B 信息地图输出格式
```
## 信息地图

### 核心事实（已交叉验证≥2源）
- [事实1] ✅ 来源A + 来源B
- [事实2] ✅ 来源C

### 待核实
- [说法1] 🟡 仅来源D提及
- [数据1] 🟡 来源E，尚未交叉

### 不同观点
- 正方观点（来源F、G）：
- 反方观点（来源H）：

### 关键缺口
- 没有人讨论的角度：[描述]
- 缺乏的数据：[描述]
```

### 5.C 信息源区分
```
🔵 一手源：公告、论文、产品、财报、原始数据
🟢 二手源：媒体报道、分析文章、博客
🟡 二手源中可信：权威媒体、专业机构
🔴 谨慎使用：社交媒体、个人推测、没有出处的说法
```

## 6. 第二道壕沟：扎深根

> **我见过太多文章是"写得很努力但不知道在说什么"——因为写的人自己也没想清楚。**

### 6.A 核心梗概协议
不给写大段的提纲。只用**300字**写出核心：

1. **一句话核心** — 这篇文章到底在说什么？（15字以内，能为写不出来的话这篇就不该写）
2. **为什么现在说** — 这件事为什么现在值得讨论？（时效性/转折点/被忽视）
3. **核心矛盾** — 最有张力的冲突是什么？（不解决这个矛盾，文章就没有驱动力）
4. **我的立场** — 我站在哪边？（有立场但公正，不是无观点的综述）
5. **最终想带给读者什么** — 看完的人会得到什么？（认知/警示/决策参考/行动指南）

### 6.B 核心矛盾发现工具
| 矛盾类型 | 模板 | 示例 |
|---------|------|------|
| 新旧冲突 | "当X已经Y，为什么Z还在A？" | "当AI能写代码时，为什么程序员还在做CRUD？" |
| 认知冲突 | "你以为X是A，其实是B" | "你以为AI是工具，其实它是你的竞争对手" |
| 结构矛盾 | "X想要A，但Y想要B" | "用户想要免费，但公司需要盈利" |
| 趋势矛盾 | "X在加速，但Y在减速" | "AI在加速进化，但人类的适应能力在下降" |
| 价值矛盾 | "X很有价值，但没人做" | "古籍数字化很有价值，但没人找到商业闭环" |

### 6.C 300字梗概成品模板
```
## 核心梗概

**一句话核心：** [15字以内]

**为什么现在说：** [2-3句，点明时机]

**核心矛盾：** [矛盾模板 + 具体阐述]

**我的立场：** [坚定且有依据的立场]

**给读者的核心交付：** [看完能带走的一个认知/工具/判断]

**文章预判结构：**
- 开头用：[具体角度，如一个场景/一个数据/一个反常识]
- 主体推进：[如何论证核心矛盾]
- 结尾落在：[给读者的最终指向]
```

## 7. 第三道壕沟：开口子

> **读者决定看不看一篇文章的时间是3秒。开头决定一切。**

### 7.A 三版开头设计
写**三个不同角度**的开头，不先评估好坏，写完了再选：

**版本A：场景式开头**
以一个具体场景/故事/案例切入，让读者"代入"。
> 格式：场景描述（2-3句）+ 转折（1句）+ 引出问题

**版本B：数据/反常识开头**
用一个反直觉的事实或数据震住读者。
> 格式：数据/事实（1句）+ 解释为什么反常识（1-2句）+ 引出正文

**版本C：观点/宣言式开头**
直接亮明立场，让认同的人继续看，不认同的人走。
> 格式：鲜明的观点（1句）+ 为什么敢这么说（1-2句）+ 展开预告

### 7.B 开头筛选框架
```
## 三版开头

### A版（场景）
[...]

### B版（数据/反常识）
[...]

### C版（观点/宣言）
[...]

---

**评估结果：** ✨ 选用 [A/B/C] 版
**理由：** [与话题匹配度/预期效果/风险评估]

**备用：** [X] 版（如果主选效果不好立即切换）
```

### 7.C 情景预案
为选定的开头，写两种情景预案：

**如果读者反应好：** 后续可以怎么延伸？（系列化/互动话题/读者调研）

**如果读者反应差：** 可能的负面反馈是什么？怎么回应？（不是所有声音都需要回应——只回应真正的误解，忽略纯粹的恶评）

## 9. 常见思维陷阱（Banned Patterns）

这些是在三道壕沟法执行过程中最常见的陷阱。主动规避。

### 9.A 信息陷阱
- **确认偏误 🪤** — 只找支持自己观点的信息，不看反方。→ **主动搜索反驳资料**
- **单一信源 🪤** — 只从一个渠道获取信息。→ **至少3个独立信源**
- **谷歌医生 🪤** — 查到的信息不等于真实的，深度不等于正确。→ **交叉验证**
- **信息过载 🪤** — 查了30个源，反而不知道该怎么看了。→ **设时间上限（30分钟搜索）**

### 9.B 逻辑陷阱
- **幸存者偏差 🪤** — 只看到成功的案例，忽略了失败的。→ **主动找失败案例**
- **后见之明 🪤** — "我早就知道了" 其实是在读到结果后才觉得明显。→ **写判断时不看后续发展**
- **滑坡谬误 🪤** — 从一个结果推到极端的可怕结局。→ **评估每一步的具体概率**
- **稻草人 🪤** — 把对方观点改成容易反驳的弱版本。→ **先准确复述对方观点**

### 9.C 表达陷阱
- **术语轰炸 🪤** — 用难词显得专业，实际是没想清楚。→ **能用简单词说清才叫真的懂了**
- **空洞结论 🪤** — "因此我们应该重视AI"——具体呢？→ **给可执行的建议**
- **自嗨 🪤** — 写自己觉得酷的，不是读者需要的。→ **每段问一次：读者需要知道这个吗？**
- **不表态 🪤** — 分析了一堆，最后什么都不说。→ **有立场，但如实标注信心度**

## 10. 输出模式选择

根据 DEPTH_LEVEL 和问题类型选择输出格式：

### 10.A 模式对照表

| 参数组合 | 输出模式 | 典型字数 |
|---------|---------|:--------:|
| W:低 D:低 | 每日早报/快速简报 | 300-800字 |
| W:中 D:中 | 公众号标准深度文 | 1500-3000字 |
| W:中 D:高 | 分析报告/观点文 | 2000-5000字 |
| W:高 D:中 | 信息整理/对比分析 | 2000-4000字 |
| W:高 D:高 | 研究报告/决策备忘录 | 5000-10000字 |
| W:中 D:顶层 | 方法论/心法 | 2000-4000字 |

### 10.B 深度文章四幕框架

当 DEPTH_LEVEL ≥ 5 时，使用四幕框架：

**第一幕：问题/现象（建立共鸣）**
- 读者为什么在乎这件事？
- 用他们熟悉的现象切入

**第二幕：拆解（讲清逻辑链条）**
- 一步一步拆解核心矛盾
- 每一层拆解都要回答"所以呢？"

**第三幕：案例/对比（用事实撑观点）**
- 正面案例 + 反面案例
- 时间维度：过去 → 现在 → 未来

**第四幕：趋势判断（给读者留下思考）**
- 不止于现状，给出前瞻判断
- 明确指出不确定性

| 信号 | 含义 |

## 12. 最终验收清单（Pre-Flight Check）

在交付任何产出之前，逐项检查。每一项必须明确通过。

### ✅ A. 信息检查
- [ ] 查了至少5个独立信源？（如果是，INFORMATION_WIDTH 达标）
- [ ] 关键事实有至少2个独立来源交叉验证？
- [ ] 区分了一手源和二手源？
- [ ] 查了反方观点？（如果有明显的反方存在）
- [ ] 标注了不确定的信息？

### ✅ B. 逻辑检查
- [ ] 一句话核心是否够短（15字以内）？
- [ ] 核心矛盾是否明确？（没有矛盾的文章不是好文章）
- [ ] 有立场吗？（不是无观点的综述）
- [ ] 逻辑链条是否有漏洞？

### ✅ C. 表达检查
- [ ] 写了3版开头？
- [ ] 选定的开头3秒内能抓住读者？
- [ ] 没有术语轰炸？
- [ ] 每段回答"所以呢"？
- [ ] 结尾有观点/判断/行动号召？

### ✅ D. 价值检查
- [ ] 读者看完能带走什么？
- [ ] 这个交付在1个月后依然有价值吗？
- [ ] 它跟已有的内容有区分度吗？（不只是重复已知观点）

### ✅ E. 机会标记
- [ ] 是否有适合🔔标记的深挖方向？

## 13. 超出范围（不是本技能的事）

三道壕沟法不适用于：
- **简单的信息查询**（今天天气、某个单词的意思——直接回答即可）
- **数学计算/事实验证**（使用专门的工具）
- **创意写作/文学创作**（需要不同的方法论）
- **技术编程实现**（使用专门的编程技能）
- **纯粹的情绪陪伴**（这是另一个维度的事）
